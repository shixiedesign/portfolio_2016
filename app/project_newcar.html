Progressive Data Disclosure

// UI/UX design for navigation in automated vehicles

To increase user comfort with artificial intelligence in automated vehicles, we propose data transparency and end-user participation in wayfinding. This is realized by accommodating users with varying degrees of trust using the following strategies:

1. a real-time percentage evaluation of the vehicle AI’s confidence to arrive on time based on incoming data
2. "squishy" data transparency
3. request end-user input for optimized parameters during route calculation

Our video below illustrates one possible interaction model for this concept - a screen interface that aims to provide a nuanced perspective on the real-time decision-making in the car's brain depending on the incoming data.

Design Features

1. Confidence Meter

The confidence meter allows users to carry out their own risk assessments. This is done by exposing the statistical uncertainties (variance) associated with each route along with the ETA, instead of only showing the ETA to users. By allowing the user to take a risk when AI's confidence of arrival on time is low, especially when little data is available, the design respects users' personalities and individual decision making.

As the car moves through geographical locations, different information becomes available or unavailable, which affects the car's ability to travel as planned. If the user prefers more control and stay informed, a fluctuation in the percentage confidence would prompt for user participation such as to dig deep to discover what update has caused it. If the user is more hands off, the car will notify the user if it's confidence level falls below a threshold, which is customizable.

2. "Squishy" Data Transparency

All data used in the calculation of AI confidence is available to the user. These data are by default visually represented as "available" (sector light lit up) or "unavailable" (sector light go off). This is the surface level transparency - intentionally unspecific in order to protect users who do not wish to "bother". But the interface is "squishy". The interface allows the user to dig deeper into the car’s decision making processes by accessing the car’s real time data collection pathways, including sensors, satellite views, network traffic data etc.

3. End-user Participation

The confidence meter and data transparency allow the user to make informed decisions about route changes during the journey.

The design also takes into consideration of allowing users to setup profiles where they choose to optimize different journey parameters such as speed, scenery, punctuality, etc. This information is used to filter down the number of route suggestions at the onset of the journey, which can easily be thousands in the context of a large city.

Conclusion

The approach proposed in this project supports the view that the difference between computer and human does not need to be hidden, but embraced. Avoiding the suspiciously empathetic AI with a soothing voice (such as Siri and Echo), we hope to build human-computer trust on a transparent ground of reason. Somewhat ironically, this also means granting people the necessary liberty to take risks if they feel lucky sometimes.
